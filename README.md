# cypress-qa-automation-demo
QA Automation Demo with Cypress: UI testing, API testing, reusable components, fixtures, and best practices.

## 🚀Objective
This project was created as a QA Automation Demo using Cypress.

The main goal is to practice and demonstrate:
- UI test automation
- API validation
- reusable test structure
- page object model
- fixtures and test data management
- automation best practices

The project simulates common QA scenarios for web applications.

## 🛠️Technologies
- Cypress
- JavaScript
- Node.js
- REST API Testing
- Fixtures
- Page Object Model (POM)
- Git / GitHub

## 👉Application Under Test
This project uses the Automation Exercise website for automation practice:

[https://www.automationexercise.com/](https://www.automationexercise.com/)

Automation Exercise is a public demo application commonly used for QA automation and testing practice.

The application was selected because it provides realistic e-commerce scenarios for UI and end-to-end testing.


## ▶️How to Run

### Install dependencies
npm install

### Open Cypress
npx cypress open

### Run tests in headless mode
npx cypress run

## 🧪Covered Scenarios

### UI Testing
- Valid login
- Invalid login
- Error message validation
- Product selection
- Checkout flow

### API Testing
- GET request validation
- Status code validation
- Response body validation

### Automation Features
- Fixtures usage
- Reusable page objects
- Custom commands
- Screenshot capture on failure
